# Safety and Trading Flow

이 문서는 Toss AI Macro Trader for macOS의 자동감시, 주문 후보 생성, 실주문 제출 흐름을 설명합니다.

## 기본 원칙

앱의 기본값은 안전 우선입니다.

- AI는 주문을 직접 강제하지 않습니다.
- 기계적 매매 공식은 신호를 만들 뿐, 안전장치를 우회하지 않습니다.
- `라이브 주문 허용`이 꺼져 있으면 완전 자동 전략도 실주문을 제출하지 않습니다.
- 자동선택 후보는 데이터 검증과 Berkshire Guard를 통과해야 완전 자동 주문까지 갈 수 있습니다.
- 조건부 또는 보류 후보는 승인 후 주문으로 낮춰집니다.

## 자동감시 루프

자동감시가 켜져 있으면 앱은 약 1분마다 다음 일을 수행합니다.

1. 자동 리밸런싱 필요 여부 확인
2. 시세와 필요한 분봉 데이터 새로고침
3. 가격 알림 조건 확인
4. 켜진 전략의 조건 확인
5. 손절/익절 조건 확인
6. 조건 충족 시 알림, 주문 후보, 로그 생성
7. 다음 감시 예정 시각 갱신

## 가격 알림

가격 알림은 주문과 분리된 가벼운 기능입니다.

- 조건: 현재가 이상 또는 이하
- 기본 만료: 생성 후 90일
- 자동감시가 켜져 있을 때만 주기적으로 확인
- 조건 충족 시 macOS 알림과 주문 로그 기록
- 동일 알림은 15분 이내 반복 알림을 막음
- 실주문을 만들지 않음

## 전략 조건 처리

전략 조건이 충족되면 전략 모드에 따라 동작이 달라집니다.

| 모드 | 동작 |
| --- | --- |
| 알림 모드 | 주문 없이 알림과 로그만 남김 |
| 승인 후 주문 | 승인 대기 주문 후보 생성 |
| AI 검토 | AI 검토가 필요한 주문 후보 생성 |
| 완전 자동 | 안전검사를 통과하면 실주문 제출 시도 |

## 실주문 제출 조건

실주문은 다음 조건을 모두 만족해야 제출을 시도합니다.

1. 토스 API 인증 정보가 저장되어 있음
2. 계좌가 선택되어 있음
3. `라이브 주문 허용`이 켜져 있음
4. 주문 후보가 차단 상태가 아님
5. 주문 수량 또는 금액이 있음
6. 국내 주식은 예산이 최소 1주 가격 이상임
7. 중복 주문 차단에 걸리지 않음
8. 네트워크/API 오류 상태가 아님
9. 시장가 주문 추가 확인 조건에 걸리지 않음
10. Berkshire Guard 미통과 자동선택 전략이 아님

## 주문 후보 생성

조건이 맞아도 대부분의 흐름은 먼저 주문 후보를 만듭니다.

주문 후보에는 다음 정보가 저장됩니다.

- 전략명
- 종목
- 매수/매도 방향
- 지정가/시장가
- 금액 또는 수량
- 기준가
- 안전 경고
- AI 검토 메모
- 생성 시각
- 상태

사용자는 주문 로그 탭에서 후보를 확인하고 다음 중 하나를 선택할 수 있습니다.

- 모의 승인
- 취소
- 실주문 제출

## 자동 종목 선택

자동 종목 선택은 거래대금 랭킹과 분봉 신호를 함께 사용합니다.

1. 공개 시장 랭킹 조회
2. 거래대금 단위 검산
3. 예산으로 1주 이상 살 수 있는지 확인
4. 여러 기계적 매매 공식으로 최근 분봉 백테스트
5. 수익률, 승률, 손실폭, 거래대금, 변동성을 점수화
6. Berkshire Guard 판정
7. 통과 후보만 완전 자동 허용

## Berkshire Guard 판정

| 판정 | 의미 |
| --- | --- |
| 통과 | 데이터와 분봉 신호가 충분하며 완전 자동 가능 |
| 조건부 | 전략 생성 가능하지만 승인 후 주문 권장 |
| 보류 | 자동주문 차단 |

주요 차단 사유:

- 시장 랭킹 데이터 검증 미통과
- 거래대금 또는 거래량 누락
- 실매수 가능 예산 없음
- 최근 분봉 매수 신호 부족
- 최근 신호 승률 낮음
- 최악 손익이 큼
- 분봉 변동성 과다
- 단기 하락 추세 강함

## 요청 한도와 장애 처리

토스 API가 429 요청 한도를 반환하면 앱은 쿨다운을 설정합니다.

- 일정 시간 새 요청을 막음
- 자동감시 다음 실행을 뒤로 미룸
- 화면에 남은 대기 시간 표시

API 오류가 발생하면 설정에 따라 자동매매를 중지할 수 있습니다.

## 실제 운영 전 체크리스트

- 토스 API 키가 올바른지 확인
- 계좌 탭에서 계좌와 보유종목이 조회되는지 확인
- 실주문 연결 점검 실행
- 라이브 주문 허용을 켜기 전 주문 후보 흐름을 먼저 확인
- 소액 단일 종목 주문 테스트
- 가격 알림과 승인 후 주문 모드로 충분히 관찰
- 완전 자동은 마지막 단계에서만 사용

## 투자 책임

이 앱은 자동매매 보조 도구입니다. 투자 손실, 주문 오류, 네트워크 장애, 데이터 지연, API 제한, 급격한 변동성에 대한 책임은 사용자에게 있습니다.
