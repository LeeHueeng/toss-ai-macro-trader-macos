## 결론

KOQUANT는 여러 투자기법 모음이라기보다, 한국 주식 실시간 자동매매를 안전하게 운영하기 위한 시스템 골격에 가깝다. 실제 전략 코드는 `minute_momentum` 샘플 하나가 핵심이며, 이 앱에는 해당 전략을 `KOQUANT 분봉 거래대금 돌파` 공식으로 재구현했다.

저장소에 별도 LICENSE 파일이 보이지 않아 코드를 직접 복사하지 않고, 공개된 전략 조건과 운영 구조를 Swift 앱에 맞게 다시 작성했다.

## 앱에 반영한 투자기법

### KOQUANT 분봉 거래대금 돌파

원본 위치:
- `src/trader/strategy/minute_momentum.py`
- `config/strategies/minute_momentum.yaml`

원본 전략의 핵심:
- 완료된 분봉만 사용한다.
- 최근 `lookback`개 봉의 평균 거래대금을 계산한다.
- 현재 봉의 거래대금이 평균 거래대금의 `volume_multiplier`배 이상인지 본다.
- 최근 구간 VWAP보다 현재 종가가 위에 있는지 본다.
- 보유 중이면 익절 또는 손절 조건을 먼저 본다.

앱 적용 방식:
- 전략관리 > 기계적 매매 > `KOQUANT 분봉 거래대금 돌파` 추가
- 기본값 버튼:
  - 평균 기간 20봉
  - 거래대금 강도 2배
  - 손절 -1%
  - 익절 1.5%
  - 쉬는 시간 1분
- Toss 분봉 데이터에는 별도 체결별 거래대금 필드가 없으므로 `종가 x 거래량`으로 거래대금을 근사한다.
- VWAP은 최근 `lookback`개 봉의 `종가 x 거래량 / 거래량 합계`로 계산한다.
- 자동 선택 모드의 후보 비교에도 포함했다.

## 운영 구조에서 반영한 관점

KOQUANT README의 핵심 흐름은 `시장 데이터 -> 전략 -> 주문 전 안전장치 -> 브로커 -> 원장 -> 대시보드 -> 운영 제어`다. 이 앱에는 다음 관점으로 맞췄다.

- 전략 신호와 실제 주문은 분리한다.
- 라이브 주문 허용이 꺼져 있으면 실제 주문은 차단한다.
- 같은 조건이 반복으로 걸리는 일을 막기 위해 쉬는 시간, 즉 쿨다운을 둔다.
- API 오류나 요청 한도 초과가 발생하면 감시/새로고침을 보수적으로 멈춘다.
- 전략 미리보기는 다음 봉 진입, 손절, 익절, 시간청산 기준으로만 추정한다.

## 앱에 직접 넣지 않은 부분

다음 항목은 KOQUANT의 시스템 기능이지만 현재 앱 목적과 증권사 API가 달라 그대로 옮기지 않았다.

- 한국투자증권 KIS REST/WebSocket 어댑터
- FastAPI 운영 서버와 React 대시보드
- PyInstaller 기반 백엔드 번들링
- SQLite 원장 스키마 전체
- 기업행동 보정, 전략 승인 기록 저장소
- 운영 토큰 기반 halt/resume API

해당 기능들은 토스증권 API와 앱 저장 구조에 맞춘 별도 구현이 필요하다.

## 초보자용 설명

`KOQUANT 분봉 거래대금 돌파`는 “갑자기 돈이 많이 몰렸고, 가격도 평균 체결가보다 위에 있다”를 보는 공식이다. 단기 추세가 붙는 순간을 잡기 위한 룰이라 급등 초입에서는 유용할 수 있지만, 뉴스성 과열이나 장 초반 변동성에서는 손절이 빠르게 날 수 있다.

그래서 이 앱에서는 기본 손절을 -1%, 익절을 1.5%로 작게 두고, 처음에는 `알림만` 또는 `물어보고 주문`으로 쓰는 것을 권장한다.
